Icons

Full set of icons for use in Home Office services. Expanding on GOV.UK guidance for icons to include interactions that allow users to better understand information.

  • Usage: internal
  • Status: experimental
  • Last updated: 02 January 2018

Use the icons

Download for prototyping kit Download for Sketch

In the prototyping kit

  1. Download the zip
  2. Unzip and move the kit-icons folder into app > assets
  3. Go to app > views > includes > head.html and link to the new icons.css file 
    <link href="/public/kit-icons/icons.css" media="screen" rel="stylesheet" type="text/css" />
  4. On a page add this code and change the class name to the icon you want 
    <span class="hod-printer side-space icon-medium"></span>

In Sketch

  1. Download the zip
  2. The download contains SVG and PNG versions of each icon
  3. Insert them into Sketch or your favourite graphics software

Interaction principles

  • The minimum size for an icon is 16px
  • If it's part of a button the hit area should be 36px
  • The icon shouldn't change colour in a hover/active/focus states, only the background.
  • In the normal state, the icon can be a different colour using the GOV.UK colour palette - both in solid and reversed out state - as long as the colour combination meets the minimum WCAG 2.0 AA colour contrast check

Content principles

  • Every icon should use a visible and audible label to avoid confusion over what it represents.
  • Align the icon in a central position in relation to any type
  • Content can sit either side of the icon

Accessibility principles

  • icons should always have a label and use aria-hidden="true"
  • if an icon can't have a label, ensure it has screenreader only support

<span class="visually-hidden">Home button</span>

User research

  • users interpret icons differently so shouldn't be used without a label
  • don't use generic icons for specific labels, for example the calendar icon for a date of birth
